Numbers

Numbers in StackLang are stored as arbitrary precision rational numbers (using GMP's mpq_class). There are, however, two flavours of numbers - exact numbers or inexact numbers. Exact numbers are displayed as fractions, and represent "exactly" the value they have, while inexact numbers are displayed as decimals, and represent "approximately" some value. Commands that take multiple numbers will produce an inxact number if an inexact number is given as any of the arguments. Additionally, some commands, like sqrt will always produce an inexact result.
To enter an exact number, optionally enter a sign, + or -, then either a decimal number or a fraction of two whole numbers. Any number of ' may be used in a number as a thousands separator, except at the beginning of the number.

The two number constants are commands. That is, they must be evaluated before they turn into a number. Thus, for the most part, they can be used as normal numbers, but they can be quoted.

An EBNF definition of a Number is given below. 

nonzero-digit = "1" | "2" | "3" | "4" | "5" | "6" | "7" | "8" | "9" ;
digit = "0" | nonzero-digit ;

separator-symbol = "'" ;

processed-nonzero-digit = {separator-symbol}, nonzero-digit, {separator-symbol} ;
processed-digit = {separator-symbol}, digit-symbol, {separator-symbol} ;

nonzero-digit-string = {processed-digit}, processed-nonzero-digit, {processed-digit} ;
digit-string = {processed-digit} ;

inexact-symbol = "~" ;
sign-symbol = "+" | "-" ;
decimal-point-symbol = "." ;
fraction-bar-symbol = "/" ;

exact-decimal = [sign-symbol], digit-string, [decimal point, digit-string] ;
exact-fraction = [sign-symbol], digit-string, fraction-bar-symbol, nonzero-digit-string ;

inexact-decimal = inexact-symbol, {separator-symbol}, [sign-symbol], digit-string, [decimal point, digit-string] ;
inexact-fraction = inexact-symbol, {separator-symbol}, [sign-symbol], digit-string, fraction-bar-symbol, nonzero-digit-string ;

exact-number = exact-decimal | exact-fraction ;
inexact-number = inexact-decimal | inexact-fraction ;

number = exact-number | inexact-number ;

Number-related Commands

Constants

euler : -> Number
Produces Euler's number.

pi : -> Number
Produces π.

Type Predicates

number? : Any -> Boolean
Produces true if given element is a number.

exact? : Any -> Boolean
Produces true if given element is an exact number.

integer? : Any -> Boolean
Produces true if given element is an integer number.

zero? : Any -> Boolean
Produces true if given element is an exact number equal to zero.

positive? : Any -> Boolean
Produces true if given element is a positive (zero not included) number.

negative? : Any -> Boolean
Produces true if given element is a negative (zero not included) number.

even? : Any -> Boolean
Produces true if given element is an even, exact, integer number.

odd? : Any -> Boolean
Produces true if given element is an odd, exact, integer number.

Type Conversion

exact-to-inexact : Number -> Number(Inexact)
Produces the inexact version of the given number.

inexact-to-exact : Number -> Number(Exact)
Produces the exact version (not) of the given number.

number-to-string : Number -> String
Produces the number as a fraction (exact number), or as a decimal with inexact symbol, but without the ... (inexact number).

string-to-number : String -> Number
Produces a number from a string as if it were read in from the interpreter. number-to-string is the exact inverse of string-to-number. Fails with a RuntimeError if string cannot be parsed.

Arithmetic

negate : Number -> Number
Produces the negative of the number.

inverse : Number -> Number
Produces one divided by the number.

increment : Number -> Number
Produces the number plus one.

decrement : Number -> Number
Produces the number minus one.

add : Number Number -> Number
Adds two numbers together.

subtract : Number Number -> Number
Subtracts the first number from the second.

multiply : Number Number -> Number
Multiplies two numbers together.

divide : Number Number -> Number
Divides the second number by the first. Will produce a syntax error when dividing by zero.

modulo : Number Number -> Number
Takes the absolute value of both numbers, and subtracts the first from the second until the second is less than the first. Then, the sign of the second is set to the sign of the first.

quotient : Number(Exact) Number(Exact) -> Number(Exact)
Produces the whole-number part of the second number divided by the first number.

remainder : Number(Exact) Number(Exact) -> Number(Exact)
Produces the remainder when two numbers are divided.

floor : Number -> Number(Exact)
Produces the floor of the number (closest integer, rounding down).

ceil : Number -> Number(Exact)
Produces the ceiling of the number (closest integer, rounding up).

round : Number -> Number(Exact)
Produces the closest integer, rounding to even.

round* : Number -> Number(Exact)
Produces the closest integer, rounding up at 0.5.

trunc : Number -> Number(Exact)
Produces the number, without any decimal digits.

abs : Number -> Number
Produces the absolute value of the number.

sign : Number -> Number
If input is positive, produces 1, else if input is negative, produces -1, else zero.

max : Number Number -> Number
Produces the larger of the two numbers.

min : Number Number -> Number
Produces the smaller of the two numbers.

gcd : Number(Exact) Number(Exact) -> Number(Exact)
Produces the greatest common divisor of two integers (always positive). If at least one of the numbers is a fraction, produces the greatest common divisor of the numerators, divided by the least common multiple of the denominators.

lcm : Number(Exact) Number(Exact) -> Number(Exact)
Produces the least common multiple of the two numbers (always positive).

numerator : Number(Exact) -> Number(Exact)
Produces the numerator of the number.

denominator : Number(Exact) -> Number(Exact)
Produces the denominator of the number.

sqrt : Number -> Number
Produces the square root of the number - inexact unless the number is a square of a rational number.

pow : Number Number -> Number
Produces the second number raised to the power of the first number.

log : Number Number -> Number
Produces the second number's logarithm, using the first number as the base. Result is inexact unless both numbers are exact, and second number is a perfect power of the base.

log-e : Number -> Number
Produces the number's natural logarithm.

log-10 : Number -> Number
Produces the number's base-10 logarithm.

log-2 : Number -> Number
Produces the number's base-2 logarithm.

Comparison

equal? : Number(Exact) Number(Exact) -> Boolean
Produces true if the number is equal to the other number.

not-equal? : Number(Exact) Number(Exact) -> Boolean
Produces true if the numbers are not equal.

less-than? : Number Number -> Boolean
Produces true if the second number is less than the first.

greater-than? : Number Number -> Boolean
Produces true if the second number is greater than the first.

less-than-equal? : Number(Exact) Number(Exact) -> Boolean
Produces true if the second number is less than or equal to the first.

greater-than-equal? : Number(Exact) Number(Exact) -> Boolean
Produces true if the second number is greater than or equal to the first.

Trigonometric Functions

sine : Number -> Number
Produces the sine of the number. Number is expected to be in radians. Output is exact only if input is exact zero.

cosine : Number -> Number
Produces the cosine of the number. Number is expected to be in radians. Output is exact only if input is exact zero.

tangent : Number -> Number
Produces the tangent of the number. Number is expected to be in radians. Output is exact only if input is exact zero.

arcsine : Number -> Number
Produces the arcsine of the number. Number is expected to be between 1 and -1. Output is between -π/2 and π/2. Output is exact only if input is exact zero.

arccosine : Number -> Number
Produces the arccosine of the number. Number is expected to be between 1 and -1. Output is between 0 and π. Output is exact only if input is exact zero.

arctangent : Number -> Number
Produces the arctangent of the number. Output is between -π/2 and π/2. Output is exact only if input is exact zero.

arctangent2 : Number Number -> Number
Produces the arctangent of the second number divided by the first number. Output is between π (inclusive) and -π (exclusive). Handles any asymptotes in that range.

cosecant : Number -> Number
Produces the cosecant of the number. Number is expected to be in radians.

secant : Number -> Number
Produces the secant of the number. Number is expected to be in radians.

cotangent : Number -> Number
Produces the cotangent of the number. Number is expected to be in radians.

arccosecant : Number -> Number
Produces the arccosecant of the number. Number is expected to be less than or equal to -1 or greater than or equal to 1. Output is between -π/2 and 0 for inputs less than or equal to -1, and between 0 and π/2 for input greater than or equal to 1.

arcsecant : Number -> Number
Produces the arcsecant of the number. Number is expected to be less than or equal to -1 or greater than or equal to 1. Output is between π/2 and π for inputs less than or equal to -1, and between 0 and π/2 for inputs greater than or equal to 1.

arccotangent : Number -> Number
Produces the arccotangent of the number. Output is between 0 and π

hyperbolic-sine : Number -> Number
Produces the hyperbolic sine of the number. Domain and range are all real numbers.

hyperbolic-cosine : Number -> Number
Produces the hyperbolic cosine of the number. Domain is all real numbers, and range is from 1 to infinity.

hyperbolic-tangent : Number -> Number
Produces the hyperbolic tangent of the number. Domain is all real numbers, and range is from negative one to positive one, not inclusive.

hyperbolic-cosecant : Number -> Number
Produces the hyperbolic cosecant of the number. Domain and range are all real numbers except zero.

hyperbolic-secant : Number -> Number
Produces the hyperbolic secant of the number. Domain is all real numbers, and range is between zero and one.

hyperbolic-cotangent : Number -> Number
Produces the hyperbolic cotangent of the number. Domain is all real numbers except zero, and range is all real numbers except those within one of zero.

hyperbolic-arcsine : Number -> Number
Produces the hyperbolic arcsine of the number. Output has the same sign as the input.

hyperbolic-arccosine : Number -> Number
Produces the hyperbolic arccosine of the number. Output is always positive.

hyperbolic-arctangent : Number -> Number
Produces the hyperbolic arctangent of the number. Output has the same sign as the input.

hyperbolic-arccosecant : Number -> Number
Produces the hyperbolic arccosecant of the number. Output has the same sign as the input.

hyperbolic-arcsecant : Number -> Number
Produces the hyperbolic arcsecant of the number. Output is always positive.

hyperbolic-arccotangent : Number -> Number
Produces the hyperbolic arccotangent of the number. Output has the same sign as the input.

Other Utilities

random : -> Number
Produces a random number. Not guarenteed to be cryptographically secure.

degrees-to-radians : Number -> Number
Input is interpreted as degrees, and is converted into radians. Output has the same sign as the input.

radians-to-degrees : Number -> Number
Input is interpreted as radians, and is converted into degrees. Output has the same sign as the input.
